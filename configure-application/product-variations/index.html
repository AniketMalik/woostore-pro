Product Variations

This section will tell you how the product variations work in the application.

Supported Variation Attributes#

The application currently support the following two product attributes. However dynamic attributes support is coming soon. You can use all types of simple and virtual products seamlessly.

Size#

This is one of the attribute that the application supports in variations. You can add any values to this attribute.

Colour#

This is the other attribute which you can use to make your variations in the application. There are some things that you need to keep in check while creating variations with the color attribute.

The application compares the value of the color attribute that you put in the product data with the color values defined in the colors.dart file. You can locate this file in woostore_pro / lib / constants / colors.dart. There are a lot of colors available in that file.

How to use colors for variations ?#

The application transforms the value of the colors got with the product data before comparing it to the names in the colors.dart file. The names get transformed like this:

"Light Blue" --> "lightblue"
"Red" --> "red"
"Dark Green" --> "darkgreen"

The application trims all the white space between the names and makes the letters all small case. Make sure to keep this in check when adding colors to the colors.dart file.

You can choose to use the names for your color attribute from those already present or add new color names if you like. Make sure you follow the How to add new colors for variations guide to add new colors.

How to add new colors for variations ?#

To add new colors to the colors.dart file you need to have both the Color Name and its associated Hex Value.

For example:

'black': '#000000',
'night': '#0C090A',
'darkslateblue': '#2B3856',
'midnightblue': '#151B54',
'navyblue': '#000080',

The Color Name must follow the above shown pattern.

For example: If you want to add Royal Blue to the colors, the name must be transformed as below:

Royal Blue --> royalblue

Finally add the Hex Value infornt of the name as shown below.

New Entry
'royalblue': '#2B60DE'

Now you can enter the the above created entry to the colors.dart file

Final Result
...
...
'royalblue': '#2B60DE',

Finally save the file. If you edit the colors.dart file while running the application, you will need to hot restart or completely stop and re run the application for the changes to take effect.

note

Support for Dynamic Attributes is in development and is coming soon to the application.
